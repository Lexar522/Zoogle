# Відкладена онлайн-оплата (LiqPay)

## Поточна поведінка (як у коді)

1. **Категорія** — на кореневому значенні опції «категорія» (`option_values`) можна ввімкнути `defer_online_payment` (адмінка Filament → значення категорії).
2. **Кошик** — якщо хоч один товар належить до гілки категорії з цим прапором на корені, `CategoryCheckoutRulesService` встановлює для всього кошика `defers_online_payment`.
3. **Чекаут** — доступний лише накладений платіж (`cod`), без переходу на LiqPay одразу після оформлення (`CheckoutController`).
4. **Замовлення** — зберігаються `orders.deferred_online_payment`, сума лишається **`orders.total`** (одне число на все замовлення).
5. **Менеджер** — у Filament «Дозволити онлайн-оплату» виставляє `online_payment_unlocked_at`, надсилається лист `OrderOnlinePaymentUnlocked`.
6. **Клієнт** — оплата через LiqPay з **`amount = order.total`** (повна сума замовлення одним платежем). Див. `App\Services\Payments\LiqPayClient::checkoutFormData`.

**Висновок для команди:** часткової оплати «спочатку без тварин» немає — LiqPay завжди бере повний `total` після розблокування.

## Майбутнє: часткова / двохетапна оплата (не реалізовано)

Якщо знадобиться оплатити спочатку лише частину позицій:

- Розділити суми: наприклад `orders.amount_payable_before_unlock`, `amount_payable_after_unlock`, або прапорці на `order_items` (`requires_manager_unlock_for_online_payment`).
- LiqPay: або два платежі з різним `order_id`/суфіксом у описі, або одне замовлення з коригуванням `total` після першого платежу (потрібен аудит і узгодження з обліком).
- Callback LiqPay (`payments.liqpay.callback`) повинен явно розрізняти частковий і фінальний платіж.

Деталі домовляти окремо; до впровадження змін у БД лишається сценарій «один total після дозволу менеджера».
